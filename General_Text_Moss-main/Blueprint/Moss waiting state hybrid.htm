设计要点

头像负责“正在工作”的动势，阶段列表负责解释等待原因。骨架区域不做气泡边框，只作为未来回答正文的低对比度占位。

头像 轻微呼吸外圈，星号小幅亮起或旋转，避免强转圈 loading。
阶段 最多 3-4 条，当前阶段高亮，其余保持灰色；收到回答后整体淡出。
回答衔接 首个 token 到达时，骨架原地替换成正式 markdown 内容。
文案 保持模糊但真实，不展示百分比，不承诺不可验证的精确进度。
