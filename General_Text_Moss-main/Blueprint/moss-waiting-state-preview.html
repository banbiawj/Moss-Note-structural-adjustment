设计要点

这个方向把“等待”拆成三个层级：头像的轻微生命感、阶段文字的可解释性、以及正文区域的低对比占位。整体比当前 ellipsis 气泡更像 Moss 正在处理上下文。

头像 轻微呼吸外圈，星号小幅亮起或旋转，避免强 loading 感。
阶段 最多 3-4 条。当前阶段高亮，已完成阶段变为柔和绿色，其余保持灰色。
正文衔接 首个 token 到达时，骨架区原地替换成正式 markdown 内容，减少跳动。
文案 保持模糊但真实，不展示百分比，不承诺不可验证的精确进度。
发送后 0-500ms 显示头像呼吸和第一条“已捕捉当前上下文”，让界面立即响应。
SSE node_end: intent 如果识别到任务类型，可把阶段文案切到“正在规划局部调整 / 正在准备知识问答”。
chat_chunk 到达 隐藏阶段列表与骨架，保留 Moss 头像，正文区域直接承接 markdown。
