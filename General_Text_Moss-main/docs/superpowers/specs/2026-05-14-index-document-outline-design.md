# Index 文档目录设计

## 背景

`index.html` 当前是 Moss 的主编辑界面：中央文档编辑面板负责 Tiptap 文档编辑，底部输入区负责 AI 对话，顶部标题区域已经有讨论选择浮层。用户希望在 index 界面右侧增加“文档目录”，并确认采用 A+C 组合方案：宽屏常驻悬浮目录，空间不足时切换为右侧抽屉。

目录框的视觉风格需要参照现有聊天/讨论选择框：半透明白底、背景模糊、细边框、轻阴影、紧凑列表、灰阶悬停和当前项高亮。

## 目标

- 从当前文档内容自动生成目录。
- 默认展示 `h1`、`h2`、`h3` 三层标题。
- 点击目录项后平滑滚动到文档中的对应标题。
- 根据文档滚动位置高亮当前标题。
- 宽屏时在编辑面板右侧显示常驻悬浮目录。
- 空间不足时显示右侧目录按钮，点击后打开目录抽屉。
- 全屏模式下优先利用正文右侧留白显示目录；留白不足时使用抽屉。
- 不改变文档正文宽度，不接入后端，不修改笔记数据结构。

## 非目标

- 不做后端目录接口。
- 不保存用户目录展开状态。
- 不支持手动编辑目录项。
- 第一版不展示 `h4`、`h5`、`h6`。
- 第一版不做复杂树线，仅用缩进和当前项高亮表达层级。

## 用户体验

### 宽屏悬浮目录

桌面宽屏下，在文档编辑面板右侧显示一个轻量目录框。目录框不占用编辑面板内部宽度，也不压缩正文。

目录框对齐规则：

- 非全屏时，目录框位于中央编辑面板右侧外部。
- 全屏时，目录框位于居中正文右侧的可用留白内。
- 当右侧剩余空间不足以容纳目录时，自动切换为目录按钮。
- 目录框最大高度随面板或视口变化，内部可独立滚动。

视觉规则：

- 外层使用现有讨论选择浮层风格：`bg-white/80`、`backdrop-blur-xl`、`border-gray-100`、`rounded-xl`、轻阴影。
- 顶部标题区显示“目录”和一个收起按钮。
- 列表行使用紧凑高度、`text-sm` 或更小字号、灰色普通态。
- 悬停态使用浅灰背景。
- 当前项使用 `bg-gray-100 text-gray-900 font-medium`，并在左侧增加细竖线或小圆点。
- 标题过长时单行省略。

### 抽屉目录

在中窄屏、移动端或右侧空间不足时，只显示一个右侧目录按钮。点击按钮后，从右侧滑出目录抽屉。

抽屉规则：

- 宽度约 `18rem`，最大宽度不超过视口宽度减去安全边距。
- 使用与悬浮目录相同的列表内容和高亮规则。
- 点击关闭按钮或抽屉外区域关闭。
- 点击目录项后滚动到目标标题；移动端点击后自动收起。
- 全屏模式不会因为打开目录而退出全屏。

## 目录数据

目录从当前 Tiptap 渲染出的文档 DOM 中提取，不依赖后端。

提取规则：

- 查询 `#document-container h1, #document-container h2, #document-container h3`。
- 每一项包含 `id`、`text`、`level`。
- 标题文本取 `textContent.trim()`。
- 空文本标题不进入目录。
- 没有 `id` 的标题需要补稳定 ID，并同步到对应 DOM/Tiptap 节点，优先复用现有顶层块 ID 维护机制。
- 文档更新、导入、AI 修改、保存前 DOM 结构维护后刷新目录。

目录为空时显示轻量空状态：“暂无目录”。

## 当前项高亮

滚动容器为现有 `#document-container`。

高亮规则：

- 监听文档滚动和窗口尺寸变化。
- 根据标题元素相对于滚动容器顶部的位置计算当前标题。
- 当前标题选择“已经经过顶部偏移线、距离顶部最近”的标题。
- 顶部偏移线建议为 `72px`，避免标题刚贴近工具栏时才切换。
- 点击目录项后立即将对应项设为当前项，并触发平滑滚动。

## 响应式策略

显示模式由实际可用空间决定，而不是只依赖固定断点。

判断规则：

- 计算目录候选位置右侧是否有至少 `260px` 可用空间。
- 有足够空间时显示悬浮目录。
- 空间不足时显示目录按钮和抽屉。
- 窗口 `resize`、全屏状态切换、编辑面板高度拖拽时重新计算。

全屏模式：

- 先计算正文最大宽度区域右侧到视口右边缘的空间。
- 空间足够时常驻显示目录。
- 空间不足时切换为抽屉。
- 目录和抽屉均避开右下角保存/退出全屏按钮。

## 前端结构

在 `index.html` 内新增以下状态和方法：

- `documentOutline`：目录项数组。
- `activeOutlineId`：当前高亮标题 ID。
- `showOutlineDrawer`：抽屉打开状态。
- `outlineMode`：`floating` 或 `drawer`。
- `outlineStyle`：悬浮目录定位样式。
- `outlineButtonStyle`：目录按钮定位样式。
- `refreshDocumentOutline()`：从文档 DOM 提取目录。
- `updateActiveOutline()`：根据滚动位置更新高亮。
- `updateOutlineLayout()`：根据视口、面板和正文位置决定模式与定位。
- `scrollToOutlineItem(item)`：滚动到目录项对应标题。
- `openOutlineDrawer()` / `closeOutlineDrawer()`：抽屉控制。

模板新增：

- 悬浮目录面板。
- 目录按钮。
- 抽屉遮罩与抽屉面板。

所有目录展示复用同一份目录项渲染逻辑，避免悬浮态和抽屉态表现不一致。

## 错误和边界处理

- 标题被删除后，目录刷新并清除无效的 `activeOutlineId`。
- AI 修改文档期间目录保持可见，但文档结构变化后重新刷新。
- 目录目标元素不存在时，点击不报错，只刷新目录。
- 标题很多时目录内部滚动，不影响文档滚动。
- 当前文档没有标题时显示空状态，不显示错误。
- 浏览器不支持某些布局计算时，回退到抽屉模式。

## 测试

手动验证：

- 普通文档包含 H1/H2/H3 时能生成层级目录。
- 点击目录项能平滑滚动到对应标题。
- 文档滚动时当前目录项高亮同步变化。
- 无标题文档显示“暂无目录”。
- 非全屏宽屏显示右侧悬浮目录。
- 非全屏窄屏显示目录按钮，按钮可打开抽屉。
- 全屏宽屏优先显示右侧悬浮目录。
- 全屏窄屏使用抽屉且不退出全屏。
- 导入新文档或 AI 修改后目录刷新。
- 保存、上传、导出、讨论选择浮层和右下角按钮仍可正常使用。

自动验证：

- 新增前端行为主要在静态 `index.html` 中，后端接口不变。
- 运行 `git diff --check` 确认无空白错误。
- 如实现中触及后端或文档解析逻辑，再运行相关 Python 单元测试。
