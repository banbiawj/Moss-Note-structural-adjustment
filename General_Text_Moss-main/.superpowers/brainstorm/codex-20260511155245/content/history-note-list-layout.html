历史会话作为“笔记库”的入口方案

这些 mockup 都把 `canvas_snapshot` 当作核心内容：卡片展示编辑器文本摘要，点击后恢复该 conversation 的文档快照和会话线程，而不是展示聊天记录。

A · 独立笔记库

复用 Blueprint 的 library 体验

主编辑器之外增加一个完整 Library 视图。适合搜索、网格/列表切换、以后加星标/文件夹。推荐作为第一版主路径。

B · 编辑器侧栏

在写作页左侧展开历史

不用离开写作上下文，切换快。但会挤压沉浸式编辑区，移动端和全屏模式的边界更复杂。

C · 覆盖层

快捷历史面板

像文件切换器一样轻量，适合最近笔记和搜索。长期管理能力弱，容易变成只读入口。

我倾向的第一版

先做 A：独立 Library 页面/视图。后端新增“列出笔记会话”和“读取某个会话快照”接口；前端点击卡片时切换 `currentConversationId`，把该 conversation 最新 checkpoint 里的 `canvas_snapshot` 放回 Tiptap 编辑器。B 或 C 可以在 A 的数据接口稳定后再作为快捷入口。
